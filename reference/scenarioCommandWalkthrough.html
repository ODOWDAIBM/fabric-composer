Walkthough of the command line for a typical Concerto Scenario

We have two people here, Mae who is the overall application architect, and Lewis a full stack developer. Lewis will take on various roles through-out this; these are indentified by for example @devops

Step1: Creating a local fabric for the team

lewis@devops $ npm install -g Concerto-cli
lewis@devops $ concerto setupTestHlf --default --profile devFabric1
Downloading Hyperledger docker images...downloaded
Setting up the images...setup
Starting images...started
Configuration for Hyperledger Fabric written to '~/.concerto-connections-profiles'

lewis@devops $ tree ~/.concerto-connections-profiles
.concerto-connections-profiles
└── devFabric1
    └── connection.json

lewis@devops $ cat ~/.concerto-connections-profiles/devFabric1/connection.json
{
   "type" : "hyperledger",
   "hfcConnection": {
     "version": "0.6.0",
     "keyvalstore
     "membershipServicesURL": ["grpc://localhost:7054"],
     "peerURL": ["grpc://localhost:7051"],
     "eventHubURL": ["grpc://localhost:7053"]
}        

lewis@devops $ cat 'Mae, I've setup a test development fabric.' | mail mae@architect

Step2: Mae updates the model for a new transaction type

(we're assuming here that two models have already been created for the model and network. The process there would be to setup the git repo and create the initial npm module.)

mae@architect $ git clone git@git.ibm.com:Blockchain-WW-Labs/AnimalTracking-Model.git
mae@architect $ # make updates to add a 'Child Birth Transaction'
mae@architect $ vi AnimalTracking-Model/models/mozart.cto
mae@architect $ git commit -a -m 'added a Child Birth Transaction'
mae@architect $ git push

mae@architect $ git clone git@git.ibm.com:Blockchain-WW-Labs/AnimalTracking-Network.git
mae@architect $ # add stub implementation of the 'Child Birth Transaction'
mae@architect $ vi AnimalTracking-Network/lib/mozart.cto.js
mae@architect $ git commit -a -m 'added a stub Child Birth Transaction'
mae@architect $ git push

mae@architect $ # Check the Travis Build results to the company's internal NPM
mae@architect $ npm publish @ibm/AnimalTracking-Model

mae@architect $ # publish stub transactions to the company's internal NPM
mae@architect $ npm publish @ibm/AnimalTracking-Network

mae@architect $ cat 'Lewis, published the network and model NPMs' > mail lewis@appdev,lewis@systemdev

Step3: Lewis implements the new transaction

(showing Lewis cloning the repos on each step - may not actually be need in practice if he's kept it)

lewis@systemdev $ git clone git@git.ibm.com:Blockchain-WW-Labs/AnimalTracking-Network.git
lewis@systemdev $ npm install
Installing dependancies....

lewis@systemdev $ vi AnimalTracking-Network/lib/mozart.cto.js
lewis@systemdev $ git commit -a -m 'added a impl of Child Birth Transaction'
lewis@systemdev $ git push

lewis@systemdev $ # run travis build and tests and publish NPM to private NPM repository

lewis@systemdev $ cat 'Lewis, please deploy the business network to our test Hyperledger fabric' > mail lewis@devops
